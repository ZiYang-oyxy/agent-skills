# 主 Agent 汇总与质检清单

在合并 subagent 结果后，至少按下面清单检查一次。

## 结构检查

- 是否先讲目标和分层，再进入软件架构、芯片架构、互联、内存、协同机制
- 是否围绕芯片软硬件系统主线展开，而不是漂移到使用场景
- 是否存在章节顺序混乱、重复定义、前后术语不一致

## 技术完整性检查

- 每个关键硬件模块是否都有相应的软件使能路径
- 软件栈是否覆盖编译器、算子库、Runtime、驱动、固件、工具链这些主要层次
- 是否同时讲清计算面、控制面、总线或 NoC、内存子系统、主机接口这些核心组成
- 是否明确区分命令流、数据流、同步点和可观测路径

## 真实性检查

- 是否存在没有来源支撑的规格、指标、带宽、时延、兼容性结论
- 是否把假设误写成事实
- 是否把“互联效率高”“访存优化好”等结论写成空话而没有机制解释

## 文风检查

- 是否出现宣传语言替代技术解释
- 是否把“高性能”“高带宽”“低时延”写成空话而没有机制说明
- 是否有大段模块罗列而没有解释数据流、控制流、仲裁、同步和协同关系

## 范围检查

- 是否意外写入训练路径、推理路径、benchmark、能力边界
- 是否意外写入部署、运维、故障定位、告警体系
- 是否把不支持的能力写成默认支持

## 图表检查

- 图名、术语、正文是否一致
- Mermaid 图是否确实提升理解，而不是重复正文
- 所有新增或修改的 Mermaid 图是否已按 `$mermaid-sop-check` 流程执行 `mmdc` 校验

## 交付检查

- 摘要或概述是否使用中文
- 最终文档是否像同一个作者完成，而不是多个章节拼接
- 是否保留了必要的“假设与缺口”说明，避免误导读者
- 是否至少有一处清楚展示软件层到芯片执行单元之间的映射关系
