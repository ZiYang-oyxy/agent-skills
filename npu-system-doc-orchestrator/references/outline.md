# NPU 芯片软硬件系统介绍默认大纲

在未给出更具体要求时，按下面的 6 章组织文档。目标是让研发读者快速建立“软件怎么组织、芯片怎么分层、数据和命令怎么流、总线与内存怎么协同”的系统认识。

## 1. 方案概述与设计目标

说明这份文档聚焦的系统边界，以及为什么需要这套 NPU 芯片软硬件系统设计。

本章必须回答：

- 文档聚焦的是芯片级、板级，还是芯片加主机接口的系统边界
- 这套设计主要优化什么：算力供给、数据搬运、访存效率、可编程性、可扩展性，还是能效
- 与 CPU/GPU/固定功能加速器相比，设计取舍是什么

建议包含：

- 一段系统总览
- 一张总览架构图
- 一张术语与边界表

## 2. 软件总体架构

按层说明软件如何把上层图或任务转成可在芯片上执行的命令、数据和调度动作。

本章必须回答：

- 编译器、算子库、Runtime、驱动、固件、工具链如何分工
- 各层的输入/输出、责任边界、关键接口是什么
- 软件如何感知和利用芯片中的计算、存储、互联资源

建议包含：

- 软件分层架构图
- 各层输入/输出表
- 软件责任边界表

## 3. 芯片计算与控制架构

按计算面和控制面说明芯片内部的组织方式。

本章必须回答：

- 计算阵列、张量或向量单元、标量控制核、DMA、调度单元如何分层
- 指令、微码、命令队列或任务描述符如何驱动计算执行
- 计算资源如何被切分、调度和同步

建议包含：

- 计算与控制分层图
- 执行资源与职责表

## 4. 总线、片上互联与外部接口

重点说明命令和数据在芯片内外如何传输，而不是只列接口名称。

本章必须回答：

- 片上总线、NoC、DMA 路径、仲裁点和带宽域如何组织
- 主机到设备之间通过 PCIe、CXL 或其他接口如何协同
- 片内、片外、主机侧三类路径的控制流和数据流如何区分

建议包含：

- 数据通路图
- 控制通路图
- 总线/互联/仲裁点表

## 5. 存储与内存层次

重点说明不同层次存储的职责、容量属性、访问模式和一致性关系。

本章必须回答：

- 寄存器、片上 SRAM、缓存、本地存储、HBM/DDR 等层次如何分工
- 数据预取、搬运、复用、回写、共享如何实现
- 内存控制器、地址空间、页表或映射机制如何影响软件设计

建议包含：

- 内存层次图
- 访问路径与数据复用表
- 地址空间与一致性说明表

## 6. 软硬件协同关键机制

把核心机制讲清楚，而不是简单重复软件和硬件章节。

本章必须回答：

- 编译结果如何映射为命令流、执行计划、内存布局和同步点
- Runtime、驱动、固件如何完成队列提交、资源管理、同步、异常和可观测性
- 软件如何绕开或缓解总线、互联、访存、同步带来的系统瓶颈

建议包含：

- 软硬件协同路径图
- 命令流/数据流/同步点表
- 关键机制与典型瓶颈表

## 默认排除项

除非用户明确要求，不要把下面内容写成主章节：

- 训练路径
- 推理路径
- 性能方法论与 benchmark 分析
- 能力边界与路线图
- 部署步骤
- 运维监控
- 故障定位
- 告警体系
- 售前商务卖点
